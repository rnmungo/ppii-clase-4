Idea principal

Tailwind propone construir componentes usando clases utilitarias.

Primero armamos el HTML semantico, despues aplicamos clases como max-w-6xl, grid, p-4, text-center o lg:grid-cols-3.

Layout y grilla

Containers, grid y breakpoints

Tailwind usa CSS Grid y Flexbox mediante clases. Los prefijos responsivos indican desde que ancho se aplica una utility.

md:grid-cols-2 lg:grid-cols-4
md:grid-cols-2 lg:grid-cols-4
md:grid-cols-2 lg:grid-cols-4
md:grid-cols-2 lg:grid-cols-4

Clases importantes

  • max-w-6xl mx-auto: centra y limita el ancho del contenido.
  • grid: activa CSS Grid.
  • grid-cols-*: define cantidad de columnas.
  • gap-4, gap-6: agregan separacion entre elementos.

Breakpoints

Los prefijos indican desde que ancho se aplica una clase.

Prefijo Ejemplo Uso
sin prefijogrid-cols-1Celulares
md:md:grid-cols-2Tablets
lg:lg:grid-cols-4Escritorio
Utility classes

Margenes, paddings, colores, flex y texto

Tailwind lleva la idea de utilities al centro del framework: la mayor parte del diseno se escribe directamente con clases.

Spacing

Formato: utilities como p-4, m-2, px-6.

Tengo p-4 m-2
  • m-4: margen en todos los lados.
  • mt-6: margen superior.
  • px-8: padding horizontal.
  • py-2: padding vertical.

Personalizacion

Modificar colores y fuentes

Tailwind se personaliza desde su configuracion. En esta pagina, los colores y fuentes salen de tailwind.config.

Paleta actual

brand.primary

brand.secondary

brand.soft

brand.text

Codigo para mostrar en clase

tailwind.config = {
  theme: {
    extend: {
      colors: {
        brand: {
          primary: "#2952e3",
          secondary: "#f97316",
          background: "#f7f8fb",
          text: "#172033"
        }
      },
      fontFamily: {
        sans: ["Inter", "system-ui", "sans-serif"]
      }
    }
  }
}
